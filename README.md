# Slim Design System (Bank)

A bank-grade Design System built with **Stencil Web Components** and **React** wrappers, powered by **design tokens**.

## What’s inside

- `packages/tokens`: semantic design tokens (CSS variables) with light/dark/high-contrast themes
- `packages/web-components`: Stencil components (framework-agnostic)
- `packages/react`: React bindings + `defineCustomElements()`
- `apps/playground-react`: a Vite React app that consumes the design system

## Architecture (professional baseline)

### Design tokens (single source of truth)

- **Source**: `packages/tokens/scripts/build-tokens.mjs`
- **CSS output**: import tokens via **stable path** `@slim-ds/tokens/tokens.css`
- **JS output** (optional tooling/docs): `@slim-ds/tokens` exports `tokens`

### Web Components (Stencil)

- Tokens are injected via `packages/web-components/src/global/global.css`
- Build automatically **syncs** tokens into `src/global/generated/tokens.css` (so Stencil can bundle it)
- Components should rely on semantic variables (e.g. `--sl-surface`, `--sl-text`) not raw hex values

### React bindings

- `packages/react/src/components.ts` is generated by Stencil React Output Target
- Storybook runs from `packages/react` and loads tokens + a11y addon via `packages/react/.storybook/*`

## Prereqs

- Node.js 18+ (recommended 20+)
- pnpm (this repo uses Corepack)

## Quickstart

```bash
corepack enable
corepack prepare pnpm@9.15.4 --activate
pnpm install
pnpm build
pnpm dev
```

## Theming

Themes are CSS-variable based:

- default (light): `:root`
- dark: `[data-theme="dark"]`
- high contrast: `[data-theme="hc"]`

Example:

```html
<html data-theme="dark">
  ...
</html>
```

## BPCE theme

This repo’s default light theme is **BPCE-inspired** (primary `#581d74`).

## Accessibility (a11y) rules of the system

- **Every interactive control must have an accessible name**
  - Visible text is best; otherwise use `aria-label` / `aria-labelledby`
  - Icon-only buttons must always have an accessible name
- **Labels + help text**
  - Inputs must be associated with `<label for=...>` or `aria-label`
  - Help/error text should be connected using `aria-describedby`
  - Errors should be announced (use `role="alert"` for error message blocks)
- **Focus**
  - All focus rings use token `--sl-focus-ring` for consistency across components
  - Do not remove outlines unless replacing with an equivalent visible focus indicator
