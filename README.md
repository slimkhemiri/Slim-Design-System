## ğŸ”§ Architecture

### Design Tokens (Single Source of Truth)

- **Source**: `packages/tokens/scripts/build-tokens.mjs`
- **CSS output**: `@slimkhemiri/tokens/tokens.css` (auto-imported by React package)
- **JS output**: `@slimkhemiri/tokens` exports `tokens` object
- **Themes**: Light (default), Dark, High Contrast

### Web Components (Stencil)

- Framework-agnostic components built with Stencil
- Tokens injected via `packages/web-components/src/global/global.css`
- Components use semantic variables (`--sl-surface`, `--sl-text`)
- Built output: Custom elements ready for any framework

### React Bindings

- `packages/react/src/components.ts` generated by Stencil React Output Target
- Exports React components + `defineCustomElements()` + `tokens`
- Full TypeScript support with type-safe props
- Storybook for component documentation

## â™¿ Accessibility

The Slim Design System follows WCAG 2.1 AA standards:

- **Keyboard Navigation**: All interactive elements are keyboard accessible
- **Screen Readers**: Proper ARIA labels and semantic HTML
- **Focus Management**: Consistent focus indicators using `--sl-focus-ring`
- **Color Contrast**: All color combinations meet AA contrast ratios
- **Reduced Motion**: Respects `prefers-reduced-motion` setting

### Guidelines

- **Every interactive control must have an accessible name**
  - Visible text is best; otherwise use `aria-label` / `aria-labelledby`
  - Icon-only buttons must always have an accessible name
- **Labels + help text**
  - Inputs must be associated with `<label for=...>` or `aria-label`
  - Help/error text should be connected using `aria-describedby`
  - Errors should be announced (use `role="alert"` for error message blocks)
- **Focus**
  - All focus rings use token `--sl-focus-ring` for consistency
  - Do not remove outlines unless replacing with an equivalent visible focus indicator
